# Experimental Mode: Visual Node-Based Concept Selector

## Vision

A visual, interactive interface where users combine different elements (models, wardrobe, poses, lighting, etc.) by clicking colorful nodes arranged in a strategic visual network. The system automatically calculates and displays multi-dimensional levels (intimacy, seduction, eroticism, professionalism, etc.) based on the selected combination.

---

## Multi-Dimensional Scales (1-25)

### Primary Scales

#### 1. **Intimacy Scale** (1-25)
Measures physical closeness, detail level, and viewer-subject connection

```

7-9:   Personal Connection (intimate portraits, close-ups, emotional engagement)
```

## Interaction Design

### Node States
1. **Inactive** - Dimmed, 60% opacity, grayscale
2. **Hovered** - Glowing border, 100% opacity, scale 1.1x
3. **Selected** - Bold border, pulsing glow, 100% opacity
4. **Compatible** - Highlighted when hovering related nodes
5. **Incompatible** - Slightly red tint when selection conflicts

- Click node ‚Üí Auto-configure related settings

- Conflicting selections show warning tooltip
- Smart recommendations appear based on selections

## Implementation Architecture

### Components to Create

1. **ExperimentalMode.tsx** - Main page component
2. **NodeCanvas.tsx** - Visual node grid component
3. **VisualNode.tsx** - Individual node component
4. **LevelIndicators.tsx** - Multi-dimensional sliders component
5. **ConfigurationDisplay.tsx** - Selected config summary
6. **NodeCalculator.ts** - Logic for calculating levels from selections

### Data Structures

```typescript
interface VisualNode {
  id: string;
  category: NodeCategory;
  name: string;
  icon: string;
  color: string;
  shape: NodeShape;
  effects: NodeEffects;
  compatibleWith: string[];
  incompatibleWith: string[];
}

interface NodeEffects {
  intimacy?: [number, number]; // Range
  seduction?: [number, number];
  eroticism?: [number, number];
  professionalism?: [number, number];
  power?: [number, number];
  abstraction?: [number, number];
  boundary?: [number, number];
}

interface CalculatedLevels {
  intimacy: number;
  seduction: number;
  eroticism: number;
  professionalism: number;
  power: number;
  abstraction: number;
  boundary: number;
}

interface ExperimentalConfig {
  selectedNodes: string[];
  calculatedLevels: CalculatedLevels;
  autoGeneratedPrompt: PromptData;
  warnings: string[];
}
```

### Algorithm: Level Calculation

```
1. Start with base levels (all at 1)
2. For each selected node:
   a. Apply node's effect ranges
   b. If multiple nodes affect same dimension, use weighted average
   c. Apply combination bonuses (certain combinations amplify effects)
3. Normalize to 1-25 range
4. Apply boundary limits (e.g., corporate environment limits max eroticism)
5. Generate warnings if conflicting selections detected
6. Return final calculated levels
```
